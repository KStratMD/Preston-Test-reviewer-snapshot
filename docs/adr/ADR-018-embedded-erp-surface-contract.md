# ADR-018 — Embedded ERP Surface Contract

**Status:** Accepted (PR 10a 2026-05-04; PR 10b adapter descriptors shipped 2026-05-23)
**Decision drivers:** Wedge differentiator vs Celigo/Boomi/MuleSoft; SOC 2
audit posture; Design Principle #1 (ERP-Native).

## Context

The wedge statement we ship to Squire executive leadership is "embedded
workflows … inside NetSuite or Business Central." Today every operator
surface lives at `https://app.suitecentral.example.com/...` — a separate
web app users must context-switch into. Finance and ops users live inside
the ERP; the context-switch loses the differentiator.

PR 10 ships the embedded surface in two phases. PR 10a (this ADR) defines
the platform-agnostic contract: iframe surface, postMessage protocol,
auth model, CSP. PR 10b adds the platform adapters (NetSuite SuiteApp
manifest + Suitelet stub; BC AL Extension + page extension).

## Decision

We adopt a **single embedded surface contract** that any module
(compliance dashboard, future PR 11 reconciliation, PR 12 lineage, PR 3C
governance approvals) can mount inside. The contract is:

1. **One iframe pattern, two platform adapters.** PR 10b ships NetSuite
   SuiteApp + BC AL Extension adapters in parallel — both wrap a common
   `public/embedded/host-reference.html`. Backs Design Principle #4
   ("Dual-ERP equal citizens") with code, not just docs.
2. **postMessage envelope is the only authenticated channel.** Every
   message is a typed envelope with origin pinning, source pinning,
   sessionId pin, csrfToken validity-window check, and bigint nonce
   replay protection. No string-prefix protocols, no inline window
   property reads.
3. **Browser JS NEVER holds a raw bearer.** Production ERP host pages
   call `host-bootstrap` server-to-server (NetSuite via `N/https`, BC
   via `HttpClient`) using a per-tenant SuiteCentral service-token. The
   browser-bearer path on the standalone reference host is dev-only,
   gated by `NODE_ENV !== 'production'` (no `||` env override).
4. **Three named auth middleware exports, NOT one conditional middleware.**
   `validateHostBootstrap` (service-token), `validateGuestContext`
   (sessionId + same-origin), `validateSessionTeardown` (sessionId match
   + same-origin). Conditional dispatch was rejected because a route
   rename would silently inherit the wrong validator.
5. **Service tokens use a hash-only validation hot path.** Raw token
   lives in SecureCredentialManager (one current row per tenant); a
   sibling `embedded_service_token_versions` table holds SHA-256 hashes
   + metadata for the multi-version overlap window. `validateToken()`
   reads only the versions table — SCM is never on the validation path,
   so overlap-window validation against an OLD token continues to work
   after SCM has been overwritten with the NEW raw token.
6. **`EmbeddedRetentionJob` is the canonical Tier-B scheduled-service
   pattern.** setInterval-based (mirrors `DisasterRecoveryService`),
   idempotent `start()`, `stop()` that awaits in-flight tick. NOT a
   CronScheduler with `pg_advisory_lock` leader election (Tier-C
   follow-up). Multi-replica safety relies on each tick query being
   idempotent.
7. **CSP `frame-ancestors` is the load-bearing gate.** Allowlist:
   `https://*.netsuite.com https://*.dynamics.com`. The bootstrap-script
   SHA-256 hash in `script-src` is build-time generated by
   `scripts/generate-embedded-csp.mjs` (prebuild + pre-commit) and
   verified by `scripts/check-embedded-csp-hash.mjs` in CI.
   `embeddedCspMiddleware` is the single source of truth for the policy
   — embedded content NEVER served via Express's default static.
8. **Adapter conformance is gated from PR 10a's merge.**
   `scripts/check-adapter-conformance.mjs` enforces that every
   `src/embedded/adapters/*.adapter.ts` has a matching named test in
   `tests/playwright/embedded/adapter-conformance.spec.ts`. PR 10a
   creates the placeholder fixture (single trivial passing test, NOT
   `test.skip`). PR 10b CANNOT drop adapters without paired tests.

## Threat model — explicit out-of-scope

The envelope protects against cross-origin attackers, in-flight replay,
stale-session replay, and token-leak via rotation. **Same-origin
script-execution attacks (XSS in the host page, malicious browser
extension with content-script access) are out of scope.** Defense at
that layer requires CSP `script-src` discipline on the host page
(NetSuite's / BC's responsibility) and standard browser-extension
threat-modeling.

## Consequences

**Positive:**
- Single contract for every embedded module — no per-module bespoke
  handshake.
- Tenant attribution flows ERP session → SuiteCentral session → audit
  log without re-authentication prompts.
- postMessage protocol typed end-to-end (host TS + guest TS share
  `PostMessageProtocol.ts`).
- Security review surface is small: three named middleware + one envelope
  validate function + one CSP policy + one retention job. Each has a
  drift tripwire test.

**Negative:**
- Operators must rotate service tokens manually via CLI for the first
  release; no UI yet (Tier-C follow-up).
- Multi-replica retention runs the cleanup queries on every replica
  (idempotent, but slightly wasteful). Leader election is Tier-C.
- Standalone host requires `NODE_ENV !== 'production'`; pre-prod
  environments wanting prod parity must set `EMBEDDED_STANDALONE_HOST=disabled`
  explicitly per environment.

**Mitigated risks:**
- A pathspec-bypass commit excluding the middleware file from a CSP
  bootstrap-script change is caught by the staged-blob pre-commit hook
  AND the CI gate.
- A future PR silently dropping the `audit-skipped-tests` placeholder
  is caught by `check-adapter-conformance.mjs` returning exit 1 if the
  spec file is missing.

## Alternatives considered

- **Single conditional `embeddedAuthMiddleware` dispatching by route
  name.** Rejected: a route rename would silently inherit the wrong
  validator. Three named exports surface the choice in the route mount.
- **CronScheduler with `pg_advisory_lock` leader election for
  retention.** Deferred to Tier-C: idempotent queries make the
  multi-replica cost a small redundant query load, not a correctness
  issue. The setInterval shape is well-trodden in the codebase
  (DisasterRecoveryService).
- **Pure JS `.mjs` rotate/revoke scripts (no ts-node).** Spec called for
  `.mjs` but the operations need DI-bound SCM. Switched to `.ts` invoked
  via `ts-node` (matches existing operational scripts like
  `seed-ai-datasets.ts`). The `.mjs` extension would have forced
  duplicating rotate/revoke logic outside the repository.

## Addendum: PR 17b — Sync Error Triage module (2026-05-08)

Extends `EmbeddedModule` enum with `'sync_error_triage'`. The new module is
the operator UI for Sync Error AI Assist — lists pending fix-suggestions,
applies accepted ones via the platform's existing connector write path
(governance preserved by routing through `ConnectorManager`, which inherits
the PR 2C `validateOutboundWrite` DLP guard), and writes audit-log entries
for accept/reject/escalate.

Mounts at `/embedded/sync-error-triage.html`. Auth via existing
`validateGuestContext` middleware (sessionId header + same-origin gate);
no new auth surface introduced.

Routes added:

- `GET /api/sync-error-assist/suggestions`
- `POST /api/sync-error-assist/suggestions/:errorRecordId/accept`
- `POST /api/sync-error-assist/suggestions/:errorRecordId/reject`
- `POST /api/sync-error-assist/suggestions/:errorRecordId/escalate`

This is a contract-level change — the EmbeddedModule enum is consumed by
typed `EmbeddedNavigationEntry` shapes and any future centralized
navigation surface. PR 10b's NetSuite/BC adapters can opt-in to mounting
the new entry.
